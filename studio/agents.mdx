# Agents

Agents are intelligent orchestrators that manage multiple playbooks simultaneously, enabling complex multi-step workflows that span different tools and data sources.

<Note>
**Think of it this way:**
- **Playbook** = Single automation task (e.g., "fetch weather")
- **Agent** = Coordinator of multiple tasks (e.g., "morning routine: fetch weather, check calendar, send daily brief")
</Note>

## What is an Agent?

An Agent is a smart container that:
- üéØ **Orchestrates Multiple Playbooks** - Run several automations as one cohesive workflow
- ‚è∞ **Scheduled Execution** - Run all playbooks at specified times or intervals
- üîÑ **Sequential or Parallel** - Control how playbooks execute (one after another or simultaneously)
- üìä **Unified Monitoring** - Track all playbook executions in one place
- ü§ñ **NIRA-Powered** - Build complex agents through conversation

## Agent vs Playbook

<CardGroup cols={2}>
  <Card title="Playbook" icon="book">

## When to Use Agents

Use Agents when you need to:

<AccordionGroup>
  <Accordion title="Coordinate Multiple Data Sources">
    **Example:** Pull data from Salesforce, enrich with market data from Finnhub, analyze with AI, and post results to Slack

    **Why Agent:** Multiple playbooks working together, each handling one data source
  </Accordion>

  <Accordion title="Execute Sequential Workflows">
    **Example:** Morning routine - check weather ‚Üí check calendar ‚Üí check emails ‚Üí generate daily brief ‚Üí send to Slack

    **Why Agent:** Playbooks need to run in specific order, passing data between steps
  </Accordion>

  <Accordion title="Recurring Multi-Step Processes">
    **Example:** Hourly: monitor system health ‚Üí check API quotas ‚Üí update dashboard ‚Üí alert if issues

    **Why Agent:** Complex workflow that repeats on a schedule
  </Accordion>

  <Accordion title="Conditional Workflow Branching">
    **Example:** Sales pipeline: check new leads ‚Üí qualify leads ‚Üí assign to reps ‚Üí notify team, but only for qualified leads

    **Why Agent:** Logic spans multiple playbooks with conditional execution
  </Accordion>
</AccordionGroup>

## Agent Configuration

### Scheduling Options

Agents can be configured with flexible scheduling:

<Tabs>
  <Tab title="Time-Based">
    Run at specific times:
    - Daily at 9:00 AM
    - Weekdays at 8:00 AM and 5:00 PM
    - Every Monday at 10:00 AM
    - First day of month at 9:00 AM

```json
{
  "schedule": {
    "type": "time",
    "time": "09:00",
    "days": ["monday", "tuesday", "wednesday", "thursday", "friday"],
    "timezone": "America/Panama"
  }
}
```
  </Tab>

  <Tab title="Interval-Based">
    Run at regular intervals:
    - Every 15 minutes
    - Every hour
    - Every 6 hours
    - Every day

```json
{
  "schedule": {
    "type": "interval",
    "interval": "1h",
    "startTime": "08:00"
  }
}
```
  </Tab>

  <Tab title="Event-Triggered">
    Run when conditions are met:
    - When webhook receives data
    - When file is uploaded
    - When threshold is exceeded

```json
{
  "schedule": {
    "type": "trigger",
    "trigger": "webhook",
    "endpoint": "/webhooks/agent-123"
  }
}
```
  </Tab>
</Tabs>

### Execution Modes

Control how playbooks within an agent execute:

<CardGroup cols={2}>
  <Card title="Sequential" icon="arrow-right">
    Playbooks run one after another

    **Best for:**
    - Dependent steps
    - Data flows between playbooks
    - Resource-intensive tasks

    **Example:** Fetch data ‚Üí Process ‚Üí Analyze ‚Üí Report
  </Card>

  <Card title="Parallel" icon="diagram-project">
    Playbooks run simultaneously

    **Best for:**
    - Independent tasks
    - Speed optimization
    - Multiple data sources

    **Example:** Check 5 APIs at once, compile results
  </Card>
</CardGroup>

### Playbook Dependencies

Define relationships between playbooks:

```json
{
  "playbooks": [
    {
      "id": "fetch-sales-data",
      "name": "Fetch Sales Data",
      "dependencies": []
    },
    {
      "id": "fetch-market-data",
      "name": "Fetch Market Data",
      "dependencies": []
    },
    {
      "id": "analyze-data",
      "name": "Analyze Combined Data",
      "dependencies": ["fetch-sales-data", "fetch-market-data"]
    },
    {
      "id": "send-report",
      "name": "Send Report to Team",
      "dependencies": ["analyze-data"]
    }
  ]
}
```

## Advanced Features

<AccordionGroup>
  <Accordion title="Conditional Execution">
    Run playbooks based on conditions:
    - Only send alerts if threshold exceeded
    - Skip steps if data is empty
    - Branch based on time of day or day of week

```json
{
  "playbook": "send-alert",
  "condition": "temperature > 100"
}
```
  </Accordion>

  <Accordion title="Data Passing Between Playbooks">
    Share data between playbooks in an agent:
    - Output from one playbook becomes input to next
    - Access shared agent context
    - Transform data between steps

```json
{
  "playbook": "analyze-data",
  "inputs": {
    "salesData": "${fetch-sales-data.output}",
    "marketData": "${fetch-market-data.output}"
  }
}
```
  </Accordion>

  <Accordion title="Error Handling & Retries">
    Configure resilient agents:
    - Retry failed playbooks automatically
    - Continue on error vs stop agent
    - Fallback playbooks
    - Error notifications

```json
{
  "errorHandling": {
    "retry": true,
    "maxRetries": 3,
    "retryDelay": "5m",
    "onError": "continue"
  }
}
```
  </Accordion>

  <Accordion title="Agent Variables">
    Define reusable variables across playbooks:
    - Environment configurations
    - Shared settings
    - Dynamic values

```json
{
  "variables": {
    "slackChannel": "#sales-team",
    "reportFormat": "detailed",
    "timezone": "America/Panama"
  }
}
```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Agent didn't run on schedule">
    **Check:**
    - Agent is not paused
    - Schedule timezone is correct
    - Schedule cron expression is valid
    - System status page for outages
  </Accordion>

  <Accordion title="One playbook failed, stopped entire agent">
    **Solution:**
    - Configure error handling to "continue" instead of "stop"
    - Add retry logic to the failing playbook
    - Review playbook logs to fix underlying issue
  </Accordion>

  <Accordion title="Playbooks running in wrong order">
    **Check:**
    - Execution mode (sequential vs parallel)
    - Playbook dependencies are configured correctly
    - Review execution logs for actual order
  </Accordion>

  <Accordion title="Agent runs too slowly">
    **Optimize:**
    - Switch independent playbooks to parallel execution
    - Reduce API call frequency in playbooks
    - Cache frequently-accessed data
    - Review tool performance
  </Accordion>
</AccordionGroup>
