The NimbleBrain Studio dashboard provides a unified view of your workspaces, servers, and deployments.

## Dashboard Layout

The Studio interface is organized into several key sections:

### Top Navigation Bar

<ResponseField name="Workspace Selector" type="dropdown">
  Switch between your workspaces. The current workspace is always displayed in the top-left corner.
</ResponseField>

<ResponseField name="User Menu" type="menu">
  Access account settings, API keys, billing, and logout options.
</ResponseField>

<ResponseField name="Search" type="input">
  Quickly find servers, workspaces, or documentation across the platform.
</ResponseField>

### Main Navigation

<Tabs>
  <Tab title="Dashboard">
    Overview of workspace activity, server health, and recent deployments
  </Tab>
  <Tab title="Servers">
    List and manage all deployed MCP servers in the current workspace
  </Tab>
  <Tab title="Registry">
    Browse and search the community MCP server registry
  </Tab>
  <Tab title="Secrets">
    Manage environment variables and API keys for your servers
  </Tab>
  <Tab title="Settings">
    Workspace configuration, tokens, and team management
  </Tab>
</Tabs>

## Dashboard Widgets

### Workspace Overview

The main dashboard displays key metrics for your current workspace:

<CardGroup cols={3}>
  <Card title="Active Servers" icon="server">
    Count of currently running MCP servers
  </Card>
  <Card title="Total Deployments" icon="rocket">
    Historical deployment count
  </Card>
  <Card title="Health Status" icon="heart-pulse">
    Aggregate health across all servers
  </Card>
</CardGroup>

### Server Status Table

View all servers in the current workspace with real-time status:

| Column | Description |
|--------|-------------|
| **Name** | Server identifier from registry |
| **Status** | Running, Scaled to Zero, Deploying, Error |
| **Health** | Health check status and response time |
| **Replicas** | Current replica count (0-N) |
| **Uptime** | Time since last deployment |
| **Actions** | Quick actions: View, Scale, Remove |

<Callout type="info">
  **Scaled to Zero**: Servers automatically scale down when not in use to save resources. They'll auto-scale up on the first request.
</Callout>

### Recent Activity Feed

Track recent events across your workspace:

- Server deployments and removals
- Scale up/down events
- Health check failures
- Configuration changes
- Secret updates

## Quick Actions

### Deploy Server

<Steps>
  <Step title="Click Deploy Button">
    From the dashboard or Servers tab, click "Deploy Server"
  </Step>
  <Step title="Search Registry">
    Browse or search for an MCP server in the registry
  </Step>
  <Step title="Configure & Deploy">
    Review requirements and click "Deploy"
  </Step>
</Steps>

### Generate Claude Config

<Steps>
  <Step title="Open Server Details">
    Click on any deployed server
  </Step>
  <Step title="Click Claude Config">
    Select "Generate Claude Desktop Config"
  </Step>
  <Step title="Copy Configuration">
    Copy the generated JSON configuration
  </Step>
</Steps>

### Create API Token

<Steps>
  <Step title="Open Settings">
    Navigate to Settings â†’ API Tokens
  </Step>
  <Step title="Generate Token">
    Click "Create New Token" and set expiration
  </Step>
  <Step title="Copy & Save">
    Copy the token immediately (it won't be shown again)
  </Step>
</Steps>

## Workspace Switching

Switch between workspaces to manage different environments:

1. Click the workspace dropdown in the top-left
2. Select a workspace from the list
3. The dashboard will update to show the selected workspace

<Callout type="tip">
  Use multiple workspaces to separate development, staging, and production environments.
</Callout>

### Health Check Status

- **Healthy**: All health checks passing
- **Degraded**: Some health checks failing
- **Unhealthy**: Critical health checks failing
- **Unknown**: Health checks not yet completed

## Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `Cmd/Ctrl + K` | Open quick search |
| `Cmd/Ctrl + D` | Go to dashboard |
| `Cmd/Ctrl + S` | Go to servers |
| `Cmd/Ctrl + R` | Go to registry |
| `Cmd/Ctrl + ,` | Open settings |
