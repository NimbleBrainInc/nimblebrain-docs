Manage your MCP servers with visual controls for deployment, scaling, and configuration.

## Deploying Servers

### From the Registry

<Steps>
  <Step title="Browse Registry">
    Navigate to the **Registry** tab to explore available MCP servers
  </Step>
  <Step title="Search & Filter">
    Use the search bar or filter by category (Data, Tools, APIs, etc.)
  </Step>
  <Step title="View Details">
    Click on a server to view:
    - Description and use cases
    - Available tools and capabilities
    - Required secrets/environment variables
    - Resource requirements
    - Community ratings and usage
  </Step>
  <Step title="Deploy">
    Click **Deploy Server** and confirm the deployment
  </Step>
</Steps>

<Callout type="info">
  Deployment typically takes 30-60 seconds. You'll see a progress indicator while the server is provisioning.
</Callout>

### Server Requirements

Some servers require configuration before deployment:

<AccordionGroup>
  <Accordion title="Secrets Required">
    Servers that integrate with external APIs (e.g., `finnhub-mcp`, `github-mcp`) need API keys configured in [Secrets Management](/studio/secrets-management) before deployment.
  </Accordion>
  <Accordion title="Environment Variables">
    Some servers accept optional configuration via environment variables. These can be set during deployment or updated later.
  </Accordion>
  <Accordion title="Resource Limits">
    Servers have default CPU and memory limits. Enterprise plans can customize these limits.
  </Accordion>
</AccordionGroup>

## Scaling Servers

### Auto-Scaling (Default)

Servers automatically scale based on demand:

- **Scale to Zero**: After 5 minutes of inactivity, servers scale down to 0 replicas
- **Scale Up**: On the first request, servers scale to 1 replica (cold start ~2-5 seconds)
- **Scale Out**: Under heavy load, servers automatically scale to multiple replicas

<Callout type="tip">
  Cold starts are typically 2-5 seconds. For latency-sensitive applications, consider using **Minimum Replicas** to keep at least one instance warm.
</Callout>

### Manual Scaling

Override auto-scaling with manual controls:

<Steps>
  <Step title="Open Server Details">
    Click on the server you want to scale
  </Step>
  <Step title="Click Scale">
    Select **Scale** from the actions menu
  </Step>
  <Step title="Set Replica Count">
    Choose:
    - **Min Replicas**: Minimum instances (0 for scale-to-zero)
    - **Max Replicas**: Maximum instances under load
  </Step>
  <Step title="Apply">
    Click **Update Scaling** to apply changes
  </Step>
</Steps>

### Scaling Examples

| Use Case | Min Replicas | Max Replicas |
|----------|--------------|--------------|
| **Development** | 0 | 1 |
| **Low Traffic Production** | 1 | 3 |
| **High Traffic Production** | 2 | 10 |
| **Demo/Testing** | 1 | 1 |

## Server Operations

### Restart Server

Force a server restart to apply configuration changes:

1. Open server details
2. Click **Actions** â†’ **Restart**
3. Confirm restart

<Callout type="warning">
  Restarting will cause brief downtime (~30 seconds) while new replicas start.
</Callout>

### Update Configuration

Modify environment variables or settings:

<Steps>
  <Step title="Navigate to Configuration">
    Open server details and click the **Configuration** tab
  </Step>
  <Step title="Edit Settings">
    Update environment variables, resource limits, or auto-scaling settings
  </Step>
  <Step title="Save Changes">
    Click **Save** - changes will be applied on next restart
  </Step>
  <Step title="Restart (if needed)">
    Some changes require a restart to take effect
  </Step>
</Steps>

### Remove Server

Permanently delete a deployed server:

1. Open server details
2. Click **Actions** â†’ **Remove Server**
3. Confirm deletion (this cannot be undone)

<Callout type="danger">
  Removing a server will delete all deployment data and configuration. Any integrations using this server will stop working.
</Callout>

## Server Health

### Health Check Types

Studio monitors server health with multiple checks:

<CardGroup cols={2}>
  <Card title="Liveness Check" icon="heartbeat">
    Ensures the server process is running
  </Card>
  <Card title="Readiness Check" icon="circle-check">
    Verifies the server can handle requests
  </Card>
  <Card title="MCP Protocol Check" icon="handshake">
    Tests MCP protocol compliance
  </Card>
  <Card title="Tool Availability" icon="wrench">
    Confirms all tools are accessible
  </Card>
</CardGroup>

### Health Status

- **Healthy**: All checks passing, server is fully operational
- **Degraded**: Some checks failing, server may have limited functionality
- **Unhealthy**: Critical checks failing, server is not operational
- **Unknown**: Health checks haven't run yet or server is scaled to zero

## Generating Claude Config

Connect your servers to Claude Desktop:

<Steps>
  <Step title="Open Server Details">
    Navigate to any deployed server
  </Step>
  <Step title="Generate Config">
    Click **Claude Desktop Config** button
  </Step>
  <Step title="Copy Configuration">
    A modal will appear with JSON configuration - click **Copy to Clipboard**
  </Step>
  <Step title="Update Claude">
    Paste into your Claude Desktop configuration file:
    - macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
    - Windows: `%APPDATA%\Claude\claude_desktop_config.json`
  </Step>
  <Step title="Restart Claude">
    Quit and restart Claude Desktop to load the new server
  </Step>
</Steps>

<Callout type="info">
  The configuration includes your workspace token for authentication. Treat it like a password!
</Callout>
