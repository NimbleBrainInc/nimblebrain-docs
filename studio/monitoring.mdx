Monitor server performance, view logs, and track metrics in real-time from NimbleBrain Studio.

## Server Monitoring Dashboard

Access monitoring for any deployed server:

<Steps>
  <Step title="Navigate to Server">
    Go to **Servers** tab and click on a server
  </Step>
  <Step title="View Metrics Tab">
    Click the **Metrics** tab to see performance data
  </Step>
</Steps>

## Real-Time Logs

### Viewing Logs

<Steps>
  <Step title="Open Server Details">
    Click on any deployed server
  </Step>
  <Step title="Navigate to Logs Tab">
    Click the **Logs** tab
  </Step>
  <Step title="View Log Stream">
    Logs stream in real-time with automatic scrolling
  </Step>
</Steps>

### Log Filtering

Filter logs by severity level:

<Tabs>
  <Tab title="All Logs">
    View all log entries (info, warn, error, debug)
  </Tab>
  <Tab title="Errors Only">
    Filter to show only ERROR level logs for debugging
  </Tab>
  <Tab title="Warnings">
    Show WARN and ERROR logs
  </Tab>
  <Tab title="Search">
    Use the search bar to filter by text content
  </Tab>
</Tabs>

### Log Features

<CardGroup cols={2}>
  <Card title="Auto-Scroll" icon="arrow-down">
    Automatically follows new log entries
  </Card>
  <Card title="Timestamps" icon="clock">
    Precise timestamps for each log entry
  </Card>
  <Card title="Color Coding" icon="palette">
    Visual severity indicators (red=error, yellow=warn)
  </Card>
  <Card title="Copy Logs" icon="copy">
    Click any log entry to copy to clipboard
  </Card>
</CardGroup>

### Downloading Logs

Export logs for analysis:

<Steps>
  <Step title="Click Export">
    From the Logs tab, click **Export Logs**
  </Step>
  <Step title="Select Time Range">
    Choose: Last hour, 24 hours, 7 days, or custom range
  </Step>
  <Step title="Choose Format">
    Download as `.txt` or `.json`
  </Step>
  <Step title="Download">
    Click **Download** to save logs locally
  </Step>
</Steps>

### Request Metrics

### Latency Metrics

Monitor response times:

| Metric | Description |
|--------|-------------|
| **p50 Latency** | Median response time (50th percentile) |
| **p95 Latency** | 95th percentile response time |
| **p99 Latency** | 99th percentile response time |
| **Max Latency** | Slowest response in time window |

<Callout type="info">
  Healthy MCP servers typically have p95 latency under 500ms. Consistently higher values may indicate performance issues.
</Callout>

    - Memory usage trend
  </Tab>
  <Tab title="Network">
    - Inbound traffic (MB/s)
    - Outbound traffic (MB/s)
    - Total data transferred
  </Tab>
</Tabs>

  </Step>

### Health Check Types

<AccordionGroup>
  <Accordion title="Liveness Probe">
    Ensures the server process is running and responsive. Failure triggers automatic restart.
  </Accordion>
  <Accordion title="Readiness Probe">
    Verifies the server is ready to accept requests. Failed checks remove the server from load balancing.
  </Accordion>
  <Accordion title="MCP Protocol Check">
    Tests MCP protocol compliance by calling `initialize` and listing tools.
  </Accordion>
</AccordionGroup>

## Alerts & Notifications

Configure alerts for important events:

### Setting Up Alerts

<Steps>
  <Step title="Navigate to Alerts">
    Go to Settings â†’ Alerts
  </Step>
  <Step title="Create Alert Rule">
    Click **+ Add Alert** and configure:
    - Alert name
    - Condition (error rate, downtime, etc.)
    - Threshold value
  </Step>
  <Step title="Set Notification Channel">
    Choose: Email, Slack, PagerDuty, or webhook
  </Step>
  <Step title="Save Alert">
    Click **Create Alert** to activate
  </Step>
</Steps>

### Common Alert Rules

<Tabs>
  <Tab title="High Error Rate">
    Trigger when error rate exceeds 5% for 5 minutes
  </Tab>
  <Tab title="Server Down">
    Alert immediately when health checks fail
  </Tab>
  <Tab title="High Latency">
    Notify when p95 latency > 1 second for 10 minutes
  </Tab>
  <Tab title="Resource Limits">
    Alert when CPU or memory usage > 90%
  </Tab>
</Tabs>

## Troubleshooting with Logs

### Common Log Patterns

<AccordionGroup>
  <Accordion title="Connection Errors">
    ```
    ERROR: Failed to connect to external API
    ERROR: ECONNREFUSED
    ```
    **Solution**: Check if required secrets are configured correctly
  </Accordion>
  <Accordion title="Authentication Failures">
    ```
    ERROR: Invalid API key
    ERROR: 401 Unauthorized
    ```
    **Solution**: Verify API key is valid and has correct permissions
  </Accordion>
  <Accordion title="Timeout Errors">
    ```
    ERROR: Request timeout after 30s
    ```
    **Solution**: External API may be slow or down. Check third-party status.
  </Accordion>
  <Accordion title="Memory Issues">
    ```
    ERROR: JavaScript heap out of memory
    ```
    **Solution**: Server may need more memory. Check resource limits in configuration.
  </Accordion>
</AccordionGroup>

### Debugging Workflow

<Steps>
  <Step title="Check Health Status">
    Start with the server health dashboard
  </Step>
  <Step title="Review Error Logs">
    Filter logs to ERROR level and search for patterns
  </Step>
  <Step title="Check Metrics">
    Look for anomalies in request rate, latency, or resource usage
  </Step>
  <Step title="Verify Configuration">
    Confirm secrets and environment variables are correct
  </Step>
  <Step title="Test Locally">
    Use [MCP Client commands](/cli/commands/mcp/connect) to test directly
  </Step>
</Steps>
