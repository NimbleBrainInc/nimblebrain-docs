`nb bundle` inspects bundles and searches the mpak registry from the command line. Two subcommands are active: `list` and `search`.

Installing and removing apps is no longer done through the CLI. Apps are workspace-scoped now, so you install them from the **Apps catalog** in the web UI (or by editing your workspace definition). The `add` and `remove` subcommands have been removed and exit with an error if you call them.

## Usage

```
nb bundle <action> [args] [flags]
```

## Actions

| Action | Description | Status |
|--------|-------------|--------|
| `list` | List configured bundles | Active |
| `search <query>` | Search the mpak registry | Active |
| `add` | Install a bundle | Removed |
| `remove` | Uninstall a bundle | Removed |

## Flags

| Flag | Description |
|------|-------------|
| `--config <path>` | Config file path |
| `--json` | Output as JSON (with `list`) |

## Listing bundles

Show the bundles recorded in your `nimblebrain.json`:

```bash
nb bundle list
```

```
Configured bundles:

  @nimblebraininc/ipinfo (named)
  @nimblebraininc/granola (named)
  ../mcp-servers/hello (local)
```

Each bundle shows its name and source type: `named` (from the mpak registry), `remote` (from a URL), or `local` (a filesystem path).

When no bundles are configured:

```bash
nb bundle list
```

```
No bundles configured.
```

### JSON output

```bash
nb bundle list --json
```

```json
[{"name":"@nimblebraininc/ipinfo"},{"name":"@nimblebraininc/granola"},{"path":"../mcp-servers/hello"}]
```

## Searching the registry

Search the mpak registry for available bundles:

```bash
nb bundle search postgres
```

```
Searching mpak registry for "postgres"...
Run: mpak search "postgres"
```

<Aside type="tip">
  For a richer search experience, use the `mpak` CLI directly: `mpak search postgres`.
</Aside>

## Installing apps

Instance-level bundle installation through the CLI has been removed. Bundles are now scoped to a workspace, so the install path lives in the product:

- **Apps catalog (recommended).** Open the web UI, go to the Apps catalog in settings, and install the app into the workspace you want it in.
- **Workspace definition.** Add the bundle to the workspace's definition directly.

The legacy `add` and `remove` subcommands still exist as stubs so old scripts fail loudly instead of silently doing nothing. They print a message pointing you to the Apps catalog and exit non-zero:

```bash
nb bundle add @nimblebraininc/postgres
```

```
Instance-level bundles have been removed. Install @nimblebraininc/postgres from the Apps catalog in settings, or add the bundle to your workspace definition.
```

```bash
nb bundle remove @nimblebraininc/granola
```

```
Instance-level bundles have been removed. Uninstall @nimblebraininc/granola from the Apps section in settings, or remove the bundle from your workspace definition.
```
